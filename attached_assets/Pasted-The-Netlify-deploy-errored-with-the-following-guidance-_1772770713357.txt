The Netlify deploy errored, with the following guidance provided:

Diagnosis
- The build step that runs "npm ci && npx vite build" failed at the npm ci step ([line 53](#L53)).  
- npm reports that package.json and package-lock.json are out of sync: specifically, the lock file contains tailwind-merge@2.6.0 but package.json requires tailwind-merge@2.6.1 ([line 56](#L56), [line 58](#L58)). The build then exits with error ([line 82](#L82)).

Cause
- npm ci enforces that package.json and package-lock.json match exactly. A dependency version change in package.json (or an out-of-date lockfile) causes npm ci to fail because the lockfile does not satisfy the package.json constraint.

Solution
1. Verify the mismatch locally
- Check both files to see the versions:
```bash
# show occurrences in both files
grep -n '"tailwind-merge"' package.json package-lock.json || true

# or use jq if installed to inspect dependencies
jq '.dependencies["tailwind-merge"], .devDependencies["tailwind-merge"]' package.json || true
jq '.packages[]? | select(.name=="tailwind-merge") | .version' package-lock.json || true
```

2. Sync the lockfile (recommended)
- Run npm install locally to update package-lock.json to match package.json, then commit the updated lockfile:
```bash
npm install
git add package-lock.json
git commit -m "Sync package-lock.json with package.json (fix tailwind-merge version)"
git push
```
- Trigger a new Netlify build. npm ci will then succeed because package.json and package-lock.json will be in sync.

3. Alternative quick fixes (less recommended)
- If you intentionally want the lockfile version (2.6.0), edit package.json to require that exact version and commit package.json and package-lock.json.
- As a last resort, change the build command to use npm install (which is permissive) instead of npm ci in netlify.toml:
```toml
[build]
  command = "npm install && npx vite build"
```
But note: using npm install on CI can lead to non-reproducible installs—it's better to fix and commit the lockfile.

After fixing and committing the lockfile (or package.json), re-run the Netlify build. The current failure is caused solely by the package.json / package-lock.json mismatch and will be resolved by syncing them.
